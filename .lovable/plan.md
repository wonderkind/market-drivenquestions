

# Fresh Project Recreation Plan

If the current backend issues persist and cannot be resolved by support, here's everything that would need to be set up in a fresh Lovable project.

---

## 1. Secrets to Configure

The following secrets need to be added (you'll need the original values):

| Secret | Purpose |
|--------|---------|
| RAPIDAPI_KEY | JSearch API for job searching and job details |
| OPENAI_API_KEY | AI analysis (via Azure OpenAI endpoint) for job analysis and profile translation |
| PROFILE_API_KEY | API key authentication for profile export/questions endpoints |

The other secrets (SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, etc.) are auto-generated by Lovable Cloud.

---

## 2. Database Tables to Recreate

Six tables need to be created with RLS policies:

| Table | Purpose | RLS |
|-------|---------|-----|
| **profiles** | User profiles (auto-created on signup via trigger) | Users see/edit own profile |
| **saved_searches** | Stores job search history per user | Users see/manage own searches |
| **favorite_jobs** | Bookmarked jobs with full job data | Users see/manage own favorites |
| **analysis_results** | AI-generated job requirement analysis profiles | All authenticated users can view/edit all |
| **jobs** | Cached job listings from JSearch API | Public read access |
| **onet_occupations** | O*NET-SOC occupation catalog (867 entries) | Public read access |

Key database objects:
- **Trigger**: `on_auth_user_created` -- auto-creates a profile row when a user signs up
- **Function**: `handle_new_user()` -- inserts into profiles table
- **Function**: `update_updated_at_column()` -- auto-updates timestamps
- **Unique index** on analysis_results for preventing duplicate profiles per user

## 3. Edge Functions to Recreate (8 total)

All have `verify_jwt = false` in config:

| Function | External API | Purpose |
|----------|-------------|---------|
| **search-jobs** | RapidAPI (JSearch) | Search jobs and store results in DB |
| **get-job-details** | RapidAPI (JSearch) | Fetch enhanced job details (highlights) |
| **analyze-jobs** | Azure OpenAI (GPT-5) | AI analysis of job requirements into license/cert/qualification/operational fit categories |
| **translate-profile** | Azure OpenAI (GPT-5) | Translate O*NET profile to local job titles |
| **get-profile-export** | None (DB only) | Export profile data in multiple formats (full/template/structured) |
| **get-profile-questions** | None (DB only) | API to query profile questions by ID or name |
| **invite-user** | None (Supabase Auth) | Admin invite users by email |
| **seed-onet-occupations** | None (DB only) | Populate 867 O*NET occupation codes |
